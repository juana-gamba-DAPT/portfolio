Project 01

Automated KPI Alert System

Gaming Industry · Tableau + Email Automation

The Problem

A core business metric required constant manual monitoring. When critical thresholds were breached, the team had no proactive mechanism to detect or escalate the issue in time.

Key Business Questions

  • Which venues are currently in a critical metric state?
  • How long have they sustained this condition?
  • What does the week-over-week trend look like?

Solution

Designed and deployed an automated alert pipeline that detects threshold breaches and delivers a structured email digest directly to stakeholders — removing the need for manual checks entirely.

Tableau Server Metric Design Email Automation KPI Monitoring Excel Business Logic
Automated email alert · triggered by metric breach detection
Week-over-week trend view for escalated venues
Venue identifiers anonymised

Project 02

Product Analytics Suite — Gaming Portfolio

Gaming Industry · Tableau · SQL / Snowflake

The Problem

The product analytics team had no centralised view of the full game portfolio. Every analytical question required a new, ad hoc study — time-consuming and inconsistent across teams.

Key Business Questions

  • How is each product segment performing vs target?
  • Which packs drive the most revenue per machine?
  • Which venues are top and bottom performers?
  • How does performance evolve month-over-month?

Solution

Built a multi-view workbook that serves as a single source of truth for the entire portfolio. Live data via Snowflake ensures the team always works with up-to-date figures — no manual refreshes needed.

Tableau SQL · Snowflake Tableau Cloud Executive Reporting Product Analytics Data Modelling Excel
Segment overview · live KPIs + trend lines
Executive view · profitability analysis + top/bottom ranking
Client and product names anonymised

Project 03

Spain Insolvency Market Intelligence

Financial Services · Python · Looker Studio

The Problem

No structured view existed of the national insolvency landscape. Strategic decisions about market targeting were made without reliable data on trends, geography or debtor profiles.

Key Business Questions

  • Which regions show the highest insolvency incidence?
  • How has the market evolved from 2021 to 2025?
  • What sectors and debtor profiles dominate?
  • What drives insolvency volume? (correlation with unemployment, closures)

Solution

Collected, cleaned and modelled 5 years of public registry data using Python and web scraping. Built an interactive report enabling geographic and segmentation analysis to support expansion decisions.

Python Web Scraping Looker Studio Data Cleaning Google Sheets Public Data Market Intelligence
Dashboard overview · key metrics + navigation
Regional distribution + insolvency type segmentation

Project 04

Operational Efficiency Monitor

Financial Services · Python · Google Data Studio

The Problem

A regional office had no real-time view of its operational health. Case progression, client status and sales closure rates were tracked manually, making it impossible to detect risk or inefficiency early.

Key Business Questions

  • How many active clients are in the pipeline — and in what stage?
  • Is the monthly sales closure rate on target?
  • Where are operational bottlenecks occurring?
  • Which acquisition channels are growing?

Solution

Designed and implemented a multi-section operational dashboard covering client status, risk indicators and channel segmentation — giving the office manager a daily pulse on key metrics.

Python Looker Studio Operational KPIs Google Sheets Pipeline Analysis Web Scraping
Client status overview + operational risk section
Client data fully anonymised
